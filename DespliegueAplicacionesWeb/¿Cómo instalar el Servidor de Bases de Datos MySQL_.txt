¿Cómo hacer un backup de todas las bases de datos MySQL?

    Lee Notas Generales
    Lee ¿Cómo instalar el Servidor de Bases de Datos MySQL?

#!/bin/bash
 
## BEGIN CONFIG ##
HOST=localhost
USER=usuario
PASS=secreto
BACKUP_DIR=Backup
## END CONFIG ##
 
if [ ! -d $BACKUP_DIR ]; then
  mkdir -p $BACKUP_DIR
fi
 
# Backup de MySQL
MYSQL_DBS=$(mysqlshow -h $HOST -u $USER -p$PASS | awk ' (NR > 2) && (/[a-zA-Z0-9]+[ ]+[|]/) && ( $0 !~ /mysql/) { print $2 }');
 
for DB in $MYSQL_DBS ; do 
  echo "* Backuping MySQL data from $DB@$HOST..."
  mysqldump -h $HOST -u $USER -p$PASS $DB > $BACKUP_DIR/mysql_$DB.sql
done

¿Cómo cambiar la codificación de una BD MySQL?

    Primero exportamos los contenidos de la base de datos:

$ mysqldump -u usuario -p ;“add-drop-table mi_bd > mi_bd.sql

    Luego cambiamos la codificación de los datos con iconv:

$ iconv -f iso-8859-15 -t utf-8 mi_bd.sql > mi_bd-utf8.sql

ISO


http://es.wikipedia.org/wiki/UTF-8

    Y la volvemos a importar en otra base de datos creada para el efecto:

$ mysql - usuario -p mi_bd_utf8 < mi_bd-utf8.sql

¿Cómo convertir una base de datos Access a MySQL?

    Instala las herramientas necesarias:

$ sudo aptitude install mdbtools

    Extrae el esquema de todas las tablas de la base de datos original:

$ mdb-schema BD.mdb mysql > BD_esquema_mysql.sql

    Un listado de todas las tablas que forma la base de datos.

$ mdb-tables -S -1 BD.mdb > BD.txt

    Extrae, ordenadamente, los datos de cada tabla, listos para insertar en nuestra futura base de datos basada en MySQL.

$ for tabla in `cat BD.txt`; do mdb-export -Q -I BD.mdb ${tabla}; done > datos.sql

    Ahora, tendrás tanto el esquema para crear la nueva base de datos, como los datos, listos para insertarlos en ella;¦

¿Cómo permitir conexiones remotas al Servidor de Base de Datos PostgreSQL?

    Lee Notas Generales
    Lee ¿Cómo instalar el Servidor de Bases de Datos PostgreSQL?
    Editamos el archivo de configuración:

$ sudo cp /etc/postgresql/8.0/main/postgresql.conf /etc/postgresql/8.0/main/postgresql.conf.backup
$ sudo vi /etc/postgresql/8.0/main/postgresql.conf

    Ahora buscamos las siguientes líneas que se encuentran comentadas:

#listen_addresses = 'localhost'

la substituimos por la siguiente línea:

listen_addresses = '*'

posteriormente buscamos la siguiente línea y le quitamos la marca de comentario:

#password_encryption = true

y nos debe quedar lo siguiente:

password_encryption = true

    Guarde los cambios realizados en el archivo.
    Reiniciamos el demonio para que los cambios surjan efecto:

$ sudo /etc/init.d/postgresql-8.0 restart

¿Cómo configurar la lista de acceso a los usuarios del Servidor de Base de Datos PostgreSQL?

    Lee Notas Generales
    Lee ¿Cómo permitir conexiones remotas al Servidor de Base de Datos PostgreSQL?
    Editamos la lista de acceso de los usuarios:

$ sudo cp /etc/postgresql/8.0/main/pg_hba.conf /etc/postgresql/8.0/main/pg_hba.conf.backup
$ sudo vi /etc/postgresql/8.0/main/pg_hba.conf

Diríjase al final del archivo donde encontrara una lista de acceso predeterminada, ahora, dependiendo de su necesidad haga lo siguiente:

    Si necesita que cualquier usuario se conecte por medio de una dirección IP en especifico, agregue al final la siguiente línea:

host all all 192.168.1.4 255.255.255.0 md5

NOTA: La dirección IP y la SubMascara de Red son ejemplos, cámbielos por datos del usuario que requiera realizar la conexión.

    Si necesita que cualquier usuario se conecte por medio de cualquier dirección IP, agregue al final la siguiente línea:

host all all 0.0.0.0 0.0.0.0 md5

    Si necesita que un usuario determinado se conecte a una base de datos determinada por medio de una dirección IP en especifico, agregue al final la siguiente línea:

host MyDataBase MyUser 192.168.1.4 255.255.255.0 md5

    Guarde los cambios realizados en el archivo.
    Reiniciamos el demonio para que los cambios surjan efecto:

$ sudo /etc/init.d/postgresql-8.0 restart

¿Cómo crear un usuario en PostgreSQL?

    Lee Notas Generales
    Los usuarios de PostgreSQL tienen un par de capacidades que definimos en su creación. Me explico: un usuario puede o no crear más usuarios y un usuario puede o no crear bases de datos. En el ejemplo que puedes ver a continuación creamos un usuario que no puede crear más usuario (no es un administrador) pero puede crear más bases de datos. El modificador -P hace que nos pregunte por el password que le asignaremos al usuario. De otra manera el usuario se creará sin password.

$ createuser -A -d -P -h host -U usuario nuevo_usuario
Enter password for user "nuevo_usuario":
Enter it again:

    Como acabo de comentar, existen usuario administradores (pueden crear otros usuarios). Como es lógico este comando debe ser ejecutado por un usuario con esa característica.

    Análogamente podemos eliminar un usuario de esta forma:

$ dropuser -h host -U usuario usuario_borrar

¿Cómo hacer un backup de todas las bases de datos de una PostgreSQL?

#!/bin/bash
 
## BEGIN CONFIG ##
HOST=localhost
BACKUP_DIR=tmp
## END CONFIG ##
 
if [ ! -d $BACKUP_DIR ]; then
  mkdir -p $BACKUP_DIR
fi
 
POSTGRE_DBS=$(psql -h $HOST -U postgres -l | awk ' (NR > 2) && (/[a-zA-Z0-9]+[ ]+[|]/) && ( $0 !~ /template[0-9]/) { print $1 }');
 
for DB in $POSTGRE_DBS ; do 
  echo "* Backuping PostgreSQL data from $DB@$HOST..."
  pg_dump -h $HOST -U postgres $DB > $BACKUP_DIR/pg_$DB.sql
